Parameter Visualization and Controls

The visualizations and controls for individual parameters are described as follows:

  • Data Histograms:

The data histograms below each parameter name and above the parameter filter sliders display frequency of occurrence of data value sub-ranges for that parameter across its full data range (displayed at ends of data slider directly below histogram). Each bar in the data frequency histogram is colored, by number points in that sub-range that are Infeasible (red), Dominated (blue) , or Best Alternatives (green). A red bar does not necessarily require that data points in that range are in violation of a threshold for that particular parameter - feasibility is assessed across all parameter thresholds and data a points are marked infeasible if any threshold is violated. A particular data sub-range for a parameter may include any or all of Infeasible (red), Dominated (blue) , and/or Best Alternatives (green) data points, and thus the histogram bar may include any or all colors, with the bar section heights proportional to the number of data points in each category.

  • Parameter Filter Sliders:

Filter sliders for individual parameters allow filtering data points with values below a lower filter value or above an upper filter value from view. Filtered data points will not be displayed in the data table, scatter plots, etc., but are still scored and ranked, unlike excluded data points. The filter slider bar range below the lower filter slider and above the upper filter slider will be marked with dark cross hatching to indicate that the values in that range have been filtered if a lower or upper filter slider is moved. Note that data filters do not affect feasibility - they are not thresholds for marking feasible/infeasible data ranges. Feasibility is determined based on thresholds set in the Parameter Definitions table on the Define page. (See Threshold Markers below for more information.)

  • Objective Arrows:

Specification of objectives allows identification of "Tradeoff" or "Pareto optimal" data points, which are colored green and considered ' Best Alternatives'. Clicking the down arrow to the left of the parameter filter slider will define a 'minimize' objective on the parameter, while clicking the up arrow to the right of the parameter filter slider will define a 'maximize' objective. The selected arrow is colored green and the filter slider bar will be colored with a gradient from green to white (left to right for minimize objectives and right to left for maximize objectives) to indicate desired values of that parameter.

  • Threshold Markers:

Data points that violate one or more of the thresholds are marked red and are considered ' Infeasible'. When thresholds are set in the Parameter Definition table on the Define page, a marker T:(value) is displayed in the appropriate location on the filter slider bar based on the threshold value and the data range below a lower threshold and/or above an upper threshold will be colored red in the filter slider bar.
